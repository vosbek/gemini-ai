Note: For modern Cursor, .mdc files inside .cursor/rules/ are preferred over a root .cursorrules file. The .md extensions in the example above should ideally be .mdc if they are intended as Cursor rule files.

Essential Configuration Files

🏗️ Basic .cursorrules (Legacy)

This is a simpler, older format. Prefer .mdc files for more features.

# You are an expert developer assistant ## Core Principles - Always provide complete, working code - Use existing patterns from the codebase - Ask clarifying questions when needed - Explain complex decisions ## Code Quality - Follow TypeScript best practices - Include proper error handling - Write meaningful tests - Optimize for readability

⚙️ Advanced .mdc Rules (Recommended)

Example: .cursor/rules/foundation.mdc

--- description: Advanced development patterns globs: ["src/**/*.ts", "src/**/*.tsx"] alwaysApply: true # Or false if it's technology-specific --- # Expert Developer Assistant ## Context Awareness - Analyze existing code patterns - Maintain consistency across files - Use @filename syntax for references ## Implementation Standards - Follow established architecture - Use proper TypeScript types - Implement comprehensive testing

Quick Setup Commands (Example)

# Initialize new project with AI-friendly structure mkdir my-ai-project && cd my-ai-project # Create essential directories mkdir -p .cursor/rules src tests docs # Initialize package.json (for Node.js projects) npm init -y # Install essential dependencies (example for a TypeScript project) npm install typescript @types/node --save-dev # Note: Original had 'npm install typescript @types/node' (implies global or wrong save type) # For linters, formatters, test runners: npm install prettier eslint jest @types/jest ts-jest --save-dev # Create basic .cursor/rules/foundation.mdc (example) echo "--- description: Core development standards globs: [\"**/*\"] alwaysApply: true --- # You are an expert developer assistant - Always provide complete, working code. - Reference existing patterns using @filename.ext. " > .cursor/rules/foundation.mdc # Setup git git init echo "node_modules/" > .gitignore echo ".DS_Store" >> .gitignore git add . git commit -m "Initial commit with AI-friendly structure and basic Cursor rule"
