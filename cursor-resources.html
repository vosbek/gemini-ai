üí° Best Practices

Documentation Integration Tips

Keep your documentation up-to-date and well-structured. Cursor works best with markdown files, API specs in OpenAPI format, and clear code comments. Organize docs in a logical hierarchy that mirrors your code structure. Use @-file referencing extensively.

Custom Mode Strategies

Create modes (or Model Presets in Cursor) for different development phases: "Planning" mode for architecture discussions, "Implementation" mode for rapid coding, "Review" mode for code analysis, and "Documentation" mode for writing docs and comments.

Rule File Management (.mdc)

Regularly review and update your generated rules in .cursor/rules/. Start with auto-generated rules as a foundation, then customize them based on team feedback and project evolution. Version control your .cursor directory.

Diagram Workflow Integration

Generate diagrams early in development to validate architecture decisions. Update diagrams when refactoring major components. Use sequence diagrams for API flows and flowcharts for business logic visualization.

üõ†Ô∏è Troubleshooting Links Not Working

Common Issues & Solutions for Navigation

If the main navigation links (Home, Mindset Shift, etc.) or other internal links are not working, consider the following:

  • Testing Environment: If you are opening HTML files directly in your browser (e.g., using file:///... URLs), the JavaScript fetch() command used in js/main.js to load shared-header.html might be blocked by browser security policies (CORS).
    • Solution: Test your website using a local web server. Many simple servers are available (like Python's http.server, Node.js's serve or live-server extension in VS Code). This serves files over http://localhost and allows fetch() to work correctly.
  • File Paths: Ensure that js/main.js, style.css, and shared-header.html are in the correct locations relative to your HTML files, as per the recommended file structure (js folder for main.js; style.css and shared-header.html in the root with other HTML pages).
    • The js/main.js script expects shared-header.html to be at the root level (fetch('shared-header.html')).
    • HTML files link to JavaScript as <script src="js/main.js"></script>.
  • JavaScript Errors: Open your browser's developer console (usually by pressing F12) and check for any errors. Errors in js/main.js could prevent the header from loading or links from being processed correctly.
  • shared-header-placeholder ID: Make sure the div `
    ` is present in each HTML file where the header should be loaded.
  • External Links vs. Internal Links:
    • The js/main.js script is designed to make *external* links (starting with http:// or https://) open in a new tab.
    • *Internal* links (like workflow.html or index.html in the navigation) should navigate within the same tab.

The sub-pages you listed (`agentic-prompts.html`, `mindset-shift.html`, `tool-setup.html`, `workflow.html`, `cursor-resources.html`, plus `index.html` and `rule-files.html`) should all be linked correctly in the `shared-header.html` file with relative paths (e.g., `href="workflow.html"`).
